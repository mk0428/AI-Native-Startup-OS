# Measurement Framework

> Every product must define its measurement framework BEFORE the first user
> arrives. Metrics chosen after launch tend to confirm success rather than
> surface failure.
>
> Complete this document during the MVP stage, before entering Launch.

---

## 1. Activation Criteria

What must a user DO to get value from this product?

```
User signs up → [does what?] → [experiences what outcome?]
```

**Activation event:** _________________________________________________

**Target activation rate:** ____% of signups within ____ days of first visit

**What counts as NOT activated:** A user who signs up but never reaches the
activation event.

## 2. Retention Benchmarks

| Timeframe | Target | What counts as "retained" |
|-----------|--------|--------------------------|
| Day 1 | ____% | |
| Day 7 | ____% | |

| Day 90 | ____% | |

**Note:** Industry benchmarks vary by product type. For a consumer tool,
Day 7 > 30% is strong. For a productivity tool, Day 30 > 40% is strong.
For a utility tool, look at task completion rate rather than daily usage.

## 4. The Sean Ellis Test

> Ask active users: *"How would you feel if you could no longer use this product?"*

| Answer | Count | Percentage |
|--------|-------|-----------|
| Very disappointed | | ____% ← Target > 40% |
| Somewhat disappointed | | |
| Not disappointed | | |
| N/A (don't use it enough) | | |

**When to run:** After you have at least 50-100 active users who have hit the
activation event.

**The threshold:** If > 40% say "very disappointed," that's a meaningful PMF
indicator. Below 40%, you have a nice-to-have, not a must-have.

## 5. The Effort Test

> Pre-product-market fit, retention requires constant intervention:
> outreach, incentives, personal follow-up, heroic founder energy.
>
> Post-product-market fit, the product starts doing that work on its own.

**Founder effort per week on retention:** ____ hours

**Trend over last 4 weeks:** [Increasing / Decreasing / Flat]

**If effort is increasing and retention is flat → something is wrong.**

## 6. False Positive Signals

Define what would LOOK like success but isn't:

| Signal | Why it's deceptive | What to check instead |
|--------|-------------------|----------------------|
| Signup spike | Launch energy, not retention | Day 7 retention rate |
| Revenue from early users | They're friends/family/early supporters | Repeat purchase rate |
| Good feedback in interviews | People are polite | Sean Ellis test score |
| High page views | Curiosity, not need | Activation rate |
| Organic social mentions | Trend-seekers, not users | Return rate after 30 days |

## 7. Kill Conditions

> Define, before launch, what data would tell us we're wrong.

**Condition 1:** If [____ metric] is below [____ target] by [____ date],
we kill this product.

**Condition 2:** If after 3 iteration cycles we see no movement toward our
retention benchmarks, we recommend pivot or kill.

**Condition 3:** If unit economics don't pencil out (CAC > LTV) within
2 months of launch, we kill.

## 8. Pivot Diagnostic (after 3+ cycles without PMF)

When 3 iteration cycles pass without meaningful PMF movement, run this:

**Question 1:** Is there a segment in the data responding differently than the rest?
→ If yes, consider narrowing focus to that segment.

**Question 2:** Is the gap between designed value and experienced value a
positioning problem or a product problem?
→ If positioning, adjust messaging. If product, adjust features.

**Question 3:** What would have to be true for this product to find genuine PMF,
and is that scenario realistic?
→ If the assumptions needed are heroic, kill. If they're achievable, pivot.

---

*Complete before launch. Revisit at each iteration cycle.*
